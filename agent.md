# Agent Instructions for @pphatdev/registry

This repository is a CLI tool (`@pphatdev/registry`) designed to instantly download and manage custom UI components and icons for frontend projects. It dynamically fetches SVG and component files instead of bundling them, and wraps them into React/Next.js or Vue/Nuxt.js components based on a configuration file (`pphatdev.json`).

## Project Structure
- `src/`: Source code for the CLI (TypeScript).
- `src/scripts/`: Scripts to build and manage the registry (e.g., `build-registry.ts`).
- `registry/`: Contains the static JSON registry (like `index.json`) that the CLI fetches data from.
- `dist/`: Compiled output generated by `tsup`.
- `schema.json`: JSON schema for the configuration file `pphatdev.json`.

## Key Technologies
- **TypeScript**: The main language used.
- **tsup**: Used for building the CLI (`npm run build:cli`).
- **tsx**: Used for executing TypeScript scripts without a separate compilation step.
- **Commander & Inquirer**: Used for building the CLI interface and handling interactive prompts.

## Commands
- `npm run dev`: Run `tsup` in watch mode.
- `npm run build:cli`: Compile the CLI code into the `dist/` folder.
- `npm run build:registry`: Rebuild the registry JSON files. Note: This generally requires a `GITHUB_TOKEN` environment variable to prevent rate limiting.
- `npm run build`: Build both the CLI and the registry.
- `npm run start`: Run the compiled CLI from `dist/index.js`.

## Development Guidelines
- Always write logic in TypeScript inside the `src/` directory.
- When modifying the CLI commands (like `init`, `add`, or `list`), ensure you update or adhere to the existing structure.
- The CLI outputs components dynamically. Make sure formatting (like XML parsing for SVGs) is preserved during transformation.
- If you need to make changes to how the registry is built, update `src/scripts/build-registry.ts` and test with a valid `GITHUB_TOKEN`.
- Do not check in personal `GITHUB_TOKEN` credentials.
